# Known Issues & Gotchas — Adamrit

Stable, load-bearing constraints to remember when working in this codebase. Not a bug list — these are things that will trip you up if you don't know them.

---

## 1. `src/pages/FinalBill.tsx` is feature-frozen

**Do not modify** `FinalBill.tsx`, its layout, the financial-summary card, the External Requisition / Investigations tabs, the Advance Payment / Yojna Bill / Final Payment / Detailed Invoice buttons, or the printed FINAL BILL preview unless the user explicitly says **"change FinalBill.tsx"** or **"modify the final bill page"**.

**Enforcement:**
- `scripts/check-finalbill-locked.cjs` and `scripts/verify-finalbill-lock.cjs` compare the file's SHA256 against a pinned hash. Build fails if drift is detected.
- A Claude Code skill (`~/.claude/skills/finalbill-locked/`) re-asserts the rule in every session.
- The locked file's snapshot is preserved at `src/pages/FinalBill.tsx.backup`.

**Why:** the page is the legal final-bill document; visual / financial changes have audit implications. Any change requires explicit author sign-off and a SHA bump.

---

## 2. Vercel `prebuild` script was removed

Commit `038c361` removed a `prebuild` script because `.vercelignore` was blocking the files it depended on. FinalBill SHA-check is now part of the standard build chain via `npm run check:finalbill`, not a separate prebuild hook. **Do not re-introduce `prebuild`** without also auditing `.vercelignore`.

---

## 3. PMJAY / MJPJAY sidebar item was lost once — guard it

Commit history `26a5595` → `727cf81` → `792bcda` shows a sequence:

1. PMJAY/MJPJAY Yojna Package Master was **added**.
2. The sidebar menu item disappeared in a later refactor.
3. It was **restored** and a Vercel redeploy was forced.

If you touch `src/components/AppSidebar.tsx`, verify the PMJAY/MJPJAY entry still resolves after your change. See [[Routes-and-Pages]] → Masters section for the route file (`PmjayMjpjayMaster.tsx`).

---

## 4. IPD-discharge AI surgery — token budget bumped

Commit `cd6208b` raised the AI surgery output `maxOutputTokens` from `800` → `2000`. If you see truncated surgery generations in the discharge summary flow, check whether the limit was reverted. Don't lower it back below 2000 unless you have a clear reason.

---

## 5. Backup files at the repo level

The pages directory contains `FinalBill.tsx.backup` and `FinancialSummary-backup.tsx`. **Keep them.** They're intentional historical snapshots, not stale debris. Do not delete in a "cleanup" pass.

---

## 6. `*-stub.tsx` variants are intentional

Components like `AddPatientDialog-stub.tsx`, `EditPatientDialog-stub.tsx`, `LegacySurgeryMigrator-stub.tsx`, `DocumentUploadDialog-stub.tsx` are deliberately kept as placeholder variants for environments where the real implementation isn't wired up. **Do not delete** as part of an unused-file cleanup.

---

## 7. Ironbark skill catalog in `CLAUDE.md`

`CLAUDE.md` at the repo root contains an auto-generated catalog of **103 harvested skills** (managed by the Ironbark learning loop). The block lives between `<!-- IRONBARK:START -->` / `<!-- IRONBARK:END -->` markers and is **regenerated on every session start** — manual edits inside the block will be wiped. Edit outside the markers freely.

Sync is automatic every 30 min from `chatgptnotes/ironbark`. Opt-out: `IRONBARK_SYNC_DISABLED=1`.

---

## 8. ESIC / CGHS / PMJAY / MJPJAY are different schemes — don't conflate

These appear throughout pages and tariff masters. Definitions live in [[hospital-abbreviations]]. The tariff structure differs per scheme; using the wrong rate file silently produces wrong bills.

---

## 9. 70+ legacy `.md` files at adamrit repo root are mostly stale

Files like `TROUBLESHOOTING_LEDGER_ERROR.md`, `FIXED_DUPLICATE_ERROR.md`, etc. are point-in-time bug-fix logs. **They were deliberately not mirrored to this vault.** If you need historical context for a specific fix, check git log first; the .md file may be older than the actual fix in code.

---

## 10. Supabase types must be regenerated by hand

`npm run generate-types` is **not** part of any automated hook. After every migration you add, run it manually and commit `src/integrations/supabase/types.ts` in the same PR. Forgetting causes silent type drift between app and DB.
