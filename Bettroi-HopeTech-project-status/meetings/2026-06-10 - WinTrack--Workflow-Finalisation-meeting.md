# WinTrack- Workflow Finalisation meeting

*2026-06-10 — [Open in Fathom](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx)*

## Meeting Purpose

[Finalize the WinTrack workflow and define the project's phased scope.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=228.0)

## Key Takeaways

  - [**Phased Scope:** The project will be delivered in two phases: Phase 1 (CRM + Flow Order Automation) and Phase 2 (Projects Automation). This de-risks the project by delivering value faster.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=576.0)
  - [**Human-in-the-Loop AI:** The system will use a confidence score for AI-generated outputs (e.g., part compatibility). If the score is below a threshold, the task is flagged for human review, ensuring accuracy.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2228.0)
  - [**Teams-Centric Collaboration:** Internal communication and approvals will be managed in Microsoft Teams. This centralizes data, providing the AI with a richer context for learning and improving.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=3161.0)
  - [**Immediate Action:** The ModAE team must provide detailed documentation (current workflows, data samples, rules) to enable the development team to create a project plan.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=3426.0)

## Topics

### Project Scope & Phasing

  - [**Goal:** Replace ModAE's Excel-based CRM with an integrated system for CRM and proposal automation.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=228.0)
  - [**Phased Approach:** The project will be delivered in two phases to manage complexity and deliver value quickly.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=576.0)
      - [**Phase 1 (Focus):**](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=576.0)
          - [Custom CRM with lead management workflow.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=595.0)
          - [Proposal automation for "Flow Orders" (Spare Parts, Services, Retrofits).](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=610.0)
      - [**Phase 2 (Future):**](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=618.0)
          - [Automation for "Projects" (Greenfield, Upgrades).](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=618.0)
  - [**Rationale:** Phase 1 is a smaller, more manageable scope that can be completed faster, providing immediate benefits.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=576.0)

### Phase 1: CRM & Lead Management

  - [**Lead Intake:** RFQs are forwarded to `sales@modae.com`.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=1433.0)
  - [**AI Processing:** The AI parses emails, extracts data, and validates for duplicates.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=927.0)
  - [**Salesperson Assignment:** The AI assigns leads based on defined rules (e.g., region-based).](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=940.0)
  - [**Human Qualification:** Leads are classified (e.g., "Green Customer") to determine pursuit strategy.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=1356.0)
  - [**CRM Registration:** An opportunity is registered in the CRM only after qualification (AI05), preventing clutter from low-potential leads.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=1154.0)
  - [**Notifications:** Internal alerts will be sent via Microsoft Teams, not WhatsApp.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=1020.0)

### Phase 1: Proposal Automation (Flow Orders)

  - [**Knowledge Base:** ModAE must provide all data for the AI to function, including:](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2091.0)
      - [Part number decodification rules](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2415.0)
      - [Datasheets for compatibility checks](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2141.0)
      - [Rate sheets for services](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2125.0)
      - [Standard T\&Cs](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2564.0)
      - ["Gold standard" proposal templates](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2515.0)
  - [**Confidence Score:** For non-deterministic tasks (e.g., part compatibility), the AI will generate a confidence score.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2228.0)
      - [**Rule:** If the score is below a threshold (e.g., 70-90%), the task is flagged for human review.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2228.0)
      - [**Learning:** The AI learns from human corrections, increasing its confidence for future similar tasks.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2359.0)
  - [**Statement of Work (SOW):** The AI will generate SOWs for service opportunities based on inputs.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2580.0)
  - [**Human-in-the-Loop:** All customer-facing actions (e.g., sending proposals) require human approval before dispatch.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=1689.0)

### System Architecture & Collaboration

  - [**Hosting:** A VPS sidecar will host the AI models (e.g., Llama, Quen) and PostgreSQL database.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2624.0)
      - [**Rationale:** This provides better control over data security and performance than API-based models, and the expected volume (est. 150 proposals/month) is low enough for a VPS.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2624.0)
  - [**Internal Collaboration:** Microsoft Teams will be the central hub for all internal communication and approvals.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=3161.0)
      - [**Rationale:** Centralizing discussions provides the AI with a richer context for learning and improving.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=3248.0)
  - [**Reporting:** Phase 1 will feature pre-configured, static dashboards. Dynamic, AI-driven reporting is a Phase 2 feature.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2001.0)
  - [**Integration:** An API link to Align Books (ModAE's financial system) will be built to sync customer and order data.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=2984.0)

### FlowAxel Project Update

  - [**Status:** The application is functional on a local VM, but a critical blocker prevents a full demo.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=4783.0)
  - [**Blocker:** The JotForm API provides an inbox URL, not a direct task link. This forces users to log into JotForm separately, creating a poor UX and preventing data pre-filling.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=4999.0)
  - [**Resolution:** The team will demo the working parts to the client (Husayfa) and explain the JotForm API dependency.](https://fathom.video/share/RzDL5goeoM7HJ8C2BBBXazVgzDBsEgnx?tab=summary&timestamp=5371.0)
