# Claudian — Claude Code inside Obsidian

This puts an AI coding/writing agent **inside the Obsidian editor**, complementing the `@brain`
Slack bot. When you open a chat inside a project folder, the agent reads that folder's
`CLAUDE.md` (the per-project "AI employee" files generated by `gen-project-claude.mjs`) and can
read/write/search notes directly in your vault. This is the video's **Layer 2 ("Soldiers")**
experience.

> Plugin: **Claudian** by YishenTu — https://github.com/YishenTu/claudian (12.5k★, MIT, v2.0.21).
> It embeds the **Claude Code CLI**, so it reuses your existing Claude Code login. No separate
> API key, and **no Anthropic API credits needed** — it runs on your Claude Code subscription
> auth (and respects the `LLM-PROVIDER-SWITCHER` toggle).

## Why it fits our setup
- **Inherits Claude Code auth** — uses the same `claude` CLI already installed on this Mac. The
  exhausted Anthropic *API* balance does not affect it (that's pay-as-you-go API; Claudian uses
  the CLI subscription path).
- **Reads per-project `CLAUDE.md`** — the files in `Adamrit/`, `NABH-SaaS/`, `Hope-Hospital/`,
  etc. become the agent's project brief automatically when you chat from inside that folder.
- **Complements, doesn't replace** — Slack `@brain` stays the team-facing RAG brain; Claudian is
  Murali's in-editor agent for drafting and editing notes.

## Requirements (already met)
- Claude Code CLI installed → `which claude` = `/Users/murali/.npm-global/bin/claude`
- Obsidian v1.7.2+ (desktop) — this vault already runs community plugins (obsidian-git, github-sync)

## Install
1. Obsidian → Settings → **Community plugins** → **Browse**
2. Search **"Claudian"** → **Install** → **Enable**
   (or the community page: https://community.obsidian.md/plugins/realclaudian)
3. Open the chat from the ribbon icon or command palette.

## First-run config
- **Claude CLI path** — leave **empty** first so Claudian auto-detects. If you see
  `spawn claude ENOENT` (common with npm-global installs), set it manually in
  **Settings → Advanced → Claude CLI path** to `/Users/murali/.npm-global/bin/claude`.
- **Node not found** — if the CLI launches but Node isn't found, add to
  **Settings → Environment → Custom variables**: `PATH=/opt/homebrew/bin` (where `node` lives).
- **Provider** — defaults to your Claude Code subscription. To route in-editor chat through a
  cheaper model, use the `use-glm` / `use-deepseek` toggles from `LLM-PROVIDER-SWITCHER.md`
  before launching Obsidian (Claudian inherits the Obsidian process environment).

## Test it
1. In Obsidian, navigate into a project folder (e.g. `Adamrit/`).
2. Open the Claudian chat and ask: *"What is this project and what's the current status?"*
3. Confirm the answer reflects `Adamrit/CLAUDE.md` (FinalBill freeze, team members, 137 routes) —
   that proves it's reading the per-project brief.

## Privacy / housekeeping
- Claudian stores session data in `vault/.claudian/` and `vault/.claude/`. **Add both to
  `.gitignore`** so chat transcripts and local keys never get committed (the vault auto-commits
  via obsidian-git every 10 min).
- Transcripts also live in `~/.claude/projects/` (outside the vault).
- Sensitive folders (`hr`, `finance`, salary notes) are already `off_limits` in the RAG brain,
  but Claudian can read any file you point it at — be deliberate about which folder you open it in.
